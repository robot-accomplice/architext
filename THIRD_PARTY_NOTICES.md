## How Architext Is Distributed

As of 1.7.6 Architext ships as native CLI binaries (built from the Rust
workspace in `crates/`) plus a viewer compiled from Rust to WebAssembly with
[Leptos](https://leptos.dev) (client-side rendering). There is no npm package
and the viewer carries no JavaScript framework dependencies. The third-party
software bundled into the distributed artifacts is the set of Rust crates the
workspace links, transitively resolved by Cargo across every shipped target
(macOS/Linux/Windows native plus `wasm32-unknown-unknown`).

`Cargo.lock` is the source of truth for the exact transitive versions in any
given build. Build- and dev-dependencies do not ship and are not attributed
here.

## License Posture

Every bundled crate is available under a permissive license. The `accepted`
allowlist in `about.toml` contains only permissive licenses; copyleft licenses
(GPL, LGPL) are deliberately omitted. For a crate offered under an `A OR B`
expression, generation elects the first accepted (permissive) license, so the
permissive arm of dual-licensed crates such as `self_cell` (Apache-2.0 OR
GPL-2.0) and `r-efi` (… OR LGPL-2.1 …) is chosen and no copyleft obligation is
triggered. If a crate ever appears that cannot be satisfied under a permissive
license, regeneration fails rather than silently shipping the obligation.
